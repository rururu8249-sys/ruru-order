# 루루동이 관리자 UI 통일 계획

## 목적

관리자 페이지별로 흩어진 제목, 헤더, 카드, 필터, 상태명, 색상을 통일한다.

현재 기준 화면은 `/admin-live` 방송 컨트롤타워다.  
다른 관리자 메뉴는 `/admin-live`의 실무형 톤에 맞춰 정리한다.

---

## 고정 사이드 메뉴명

1. 방송
2. 주문관리
3. 입금확인
4. 고객관리
5. 정산통계
6. 설정

`정산`은 `정산통계`로 통일한다.

---

## 페이지 헤더 고정 구조

모든 관리자 페이지는 아래 구조를 따른다.

1. 작은 라벨: 루루동이 관리자
2. 큰 제목: 메뉴명
3. 설명: 이 페이지에서 하는 일 한 줄
4. 요약 카드: 3~5개
5. 주요 버튼: 새로고침 / 내보내기 / 설정 이동 등

---

## 페이지별 제목/설명

### 방송
라이브 방송 화면과 실시간 주문·입금·고객이슈를 한 화면에서 확인합니다.

### 주문관리
주문상세, 입금상태, 배송상태, 메모를 빠르게 확인하고 처리합니다.

### 입금확인
입금내역을 주문과 비교해 자동·수동 입금확인을 처리합니다.

### 고객관리
고객 정보, 주문이력, 특이사항, 차단 여부를 확인합니다.

### 정산통계
방송별 매출, 입금, 배송비, 환불, 순매출 흐름을 확인합니다.

### 설정
방송, 주문서, 배송비, 계좌, 알림 설정을 관리합니다.

## 색상 통일

- 파랑: 기본 선택, 정보 확인, 수동입금확인
- 초록: 자동입금확인, 정상 완료, 출고완료
- 빨강: 미입금, 금액불일치, 위험
- 주황: 입금확인 필요, 주의
- 노랑: 미매칭입금, 보류
- 보라: 카드결제완료
- 회색: 취소, 환불, 비활성, 숨김
- 남색/하늘: 출고대기

## 절대 건드리지 말 것

- 자동입금확인 조건
- 수동입금매칭 API
- 주문금액 계산
- 배송비 계산
- 정산 계산
- 기존 주문 데이터
- 기존 입금 데이터

자동입금확인은 닉네임 완전일치 + 금액 완전일치 + 1:1 단일 후보일 때만 허용한다.

---

## 1차 적용 방식

바로 전체 UI를 갈아엎지 않는다.

1. 기준 파일 생성
2. 입금확인 화면부터 헤더/상태명/색상 통일
3. build 검수
4. 로컬 확인
5. 주문관리로 확장
