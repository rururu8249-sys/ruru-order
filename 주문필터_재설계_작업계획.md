# 실시간 주문서 필터 재설계 — 작업계획서

작성일: 2026-06-16 · 상태: **검토 대기 (코드 미수정)**

---

## 1. 목표 (한 줄)
주문 필터를 **기간 / 범위 / 상태** 3개 축으로 정리해서, 지금 겹쳐 보이는 "방송"과 "날짜"를 깔끔하게 분리한다.

---

## 2. 지금 왜 헷갈리는지 (★ 핵심 발견)
오빠가 "방송 리스트랑 날짜가 중복 같다"고 한 게 **정확했어요.** 실제 코드를 보니 원인이 명확해요:

지금 **방송 드롭다운 한 칸에 두 종류가 섞여 있어요.**
- `현재 방송`, `전체보기` — 진짜 필터
- `0613(토) 공구·상시 주문`, `0612(금) 공구·상시 주문` … — 이건 **방송이 아니라 "쇼핑몰 상시주문"을 날짜별로 묶은 항목**이에요 (코드상 `always:2026-06-13` 같은 값).

즉 방송 드롭다운 안에 **날짜 개념(상시주문 날짜별)이 들어가 있어서** 옆의 날짜 필터랑 겹쳐 보인 거예요. 이걸 "범위" 축으로 빼내면 깔끔해져요.

그리고 `직접 선택`(custom)과 `연·월 선택`(yearmonth)은 둘 다 "특정 기간"이라 **중복이 맞아요.** 하나로 합칠 수 있어요.

---

## 3. 바꿀 것 — 3칸 구조

```
[1] 기간    전체 / 오늘 / 어제 / 최근7일 / 이번달 / 지난달 / 기간 선택
            └ "직접 선택" → "기간 선택"으로 이름 변경
            └ "연·월 선택"은 삭제 (이번달·지난달·기간선택으로 충분히 커버)

[2] 범위    전체 / 방송 주문 / 쇼핑몰(상시) 주문
            └ '방송 주문' 선택 → 위에서 고른 기간 안의 방송 리스트를 칩으로 표시 → 클릭해 좁히기
            └ 지금 방송 드롭다운에 섞여있던 '상시주문 날짜별'은 '쇼핑몰(상시)'+기간으로 자연 대체

[3] 상태    현행 그대로 유지 (전체/결제완료/입금대기/매칭필요/입금확인/카드미결제/카드결제완료/주문서취소/출고완료)
```

> 참고: 코드에 이미 "선택한 기간 안에 시작된 방송만 보여주는" 계단식 로직(`broadcastOptions`, 977줄)이 들어 있어요. 오빠 아이디어가 절반은 이미 구현돼 있어서, 이걸 "범위" 축으로 끌어내 정리하는 작업이에요.

## 4. ★ 절대 안 건드리는 것 (돈·정산 보호)
- `isPaid` (146줄), `matchesStatus` (149줄) — **결제완료/입금확인 판정 로직 무수정**
- 입금·정산·Bankda 자동입금확인·포인트 관련 전부 무수정
- 결제완료(카드+무통장)·입금확인(무통장만) 라벨은 **이미 정합** → 안 건드림
- `order_items` 신규 write 없음

이번 작업은 **"필터 값 → 어떤 주문을 보여줄지" 화면 거르기 로직만** 재배선. 돈 계산엔 손대지 않음.

## 6. ⚠️ 위험 분석 (미리 점검)
1. **`needsFullLoad` 분기 (676줄)** — 지금 `date === "yearmonth"`일 때 전체주문(1000행 페이지네이션)을 불러와요. `yearmonth`를 없애면 이 조건을 "기간 선택" 등으로 **반드시 다시 연결**해야 함. 안 그러면 과거 주문이 500건까지만 보임 → **주문 누락처럼 보이는 사고 위험.**
2. **첫 로드 기본필터 (899줄)** — 방송 중이면 자동으로 "현재 방송"으로 맞추는 로직. 범위 축 추가 시 기본값 호환 유지 필요.
3. **타입 변경 영향** — `filterYear`/`filterMonth` 제거 시 이걸 참조하는 곳(useEffect 의존성 등) 전수 확인 후 제거.

→ 이 3개는 코드 들어가기 전에 한 번 더 실제 라인 확인하고, 한 군데씩 최소 수정으로 진행.
